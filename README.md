# 📰 Daily Card News — 데일리 브리프

매일 아침, 다양한 분야의 뉴스를 큐레이션해 **카드뉴스 형태의 HTML**로 발행하는 프로젝트입니다.
AI 멀티 에이전트 파이프라인("하네스" 구조)으로 제작되며, GitHub에 매일 아카이빙됩니다.

**🔗 Repo**: [nonojin99/Daily-Card-News](https://github.com/nonojin99/Daily-Card-News)

---

## 발행 체계

매일 **2개**의 브리프가 발행됩니다.

### 1. 메인 브리프 (매일)
그날의 뉴스를 **화제성 우선**으로 선별한 종합 브리프.
분야 수는 고정하지 않고 그날의 뉴스 밀도에 따라 4~10개 카드로 유동 구성합니다.

### 2. 서브 브리프 (요일 로테이션)
한 분야를 깊게 다루는 전문 브리프. 화제성으로 거르지 않고 그 분야를 고르게 소개합니다.

| 요일 | 주제 | 내용 |
|------|------|------|
| 월 | AI·테크 | 주말새 쌓인 테크 소식 |
| 화 | 스포츠 | 경기 결과·순위·프리뷰 (KBO 제외) |
| 수 | 문화 | 도서·영화·공연·전시 |
| 목 | 음악 | K-POP·POP·클래식 + 감상 포인트 |
| 금 | 여행 | 여행지·항공권·축제 (주말 직전) |
| 토 | 액티비티·쇼핑 | 주말 행사·핫플·핫딜 |
| 일 | 우주·과학 | 한 주제 깊이 파는 롱리드 |

## 제작 파이프라인 — "하네스" 구조

```
리서치 하네스 (Haiku)   →  분야별 정보 수집·요약
편집자 하네스 (Haiku)   →  검수·팩트체크·분량 조정
마케팅 하네스 (Sonnet)  →  "마케터의 시선" 인사이트 부착
디자인 하네스 (Sonnet)  →  카드뉴스 HTML 렌더링
```

## 큐레이션 규칙

수 주간의 운영을 거치며 확립된 규칙입니다.

1. **화제성 우선** — 억지로 모든 분야를 채우지 않는다. 약한 분야는 과감히 제외.
2. **한 사건당 최대 3개 카드** — 빅이슈라도 갈래는 3개에서 끊는다.
3. **지속 정보는 변화가 있을 때만 재등장** — 전시·축제·시즌 정보는 종료 임박, 연장, 매진 등 새 변화가 있을 때만 다시 다룬다. 어제와 동일한 내용의 반복 금지.
4. **광범위 스캔 먼저** — 추적 중인 스토리에 매몰되지 말고, 매일 일반 뉴스를 넓게 훑어 새 대형 사건(재난 등)을 놓치지 않는다.
5. **오래된 뉴스를 새것처럼 다루지 않는다** — 날짜 확인이 안 되면 넣지 않는다.
6. **이미지·위성사진은 검증된 출처만** — 재난 이미지는 오정보가 흔하므로 Maxar Open Data, NASA Earth Observatory, Copernicus EMS 등 공식 출처만 신뢰.
7. **재난·참사에는 절제된 톤** — 마케팅 인사이트도 "침묵할 줄 아는 것"을 원칙으로.
8. **변동 정보에는 경고 표기** — 가격·일정·환율 등은 "⚠ 변동 가능, 링크 확인"을 명시.

### 서브 브리프 추가 규칙
- 각 항목에 **디테일 포인트**를 담는다: 작가·배경 소개, 관람 포인트, "알고 들으면 달라지는" 감상 포인트 등 (예: 볼레로의 스네어 드럼처럼, 알고 나면 다르게 보이는 지점).

## 파일 규칙

```
YYYY-MM-DD_daily-brief.html      메인 브리프
YYYY-MM-DD_music-brief.html      서브 브리프 (music/culture/sports/travel/...)
```

- 날짜(YYYY-MM-DD)를 앞에 두어 폴더에서 자동 정렬
- 파일이 쌓이면 월별 폴더로 정리 권장: `2026-06/`, `2026-07/` ...

## 운영 워크플로

1. **생성**: 매일 아침 Claude에게 "브리핑해줘" 요청 → 당일 뉴스 조사 → HTML 생성
2. **업로드**: 로컬 repo 폴더(`C:\Python Work\Daily-CARD-NEWS`)에 저장 후
   ```bash
   git add .
   git commit -m "YYYY-MM-DD daily brief"
   git push
   ```
3. **자동화(예정)**: Claude Cowork 스케줄 작업(매일 오전 7시)으로 생성→커밋→푸시 자동화

> GitHub 인증은 Git Credential Manager(브라우저 로그인)로 최초 1회 설정 완료.
> 이후 push는 무인증으로 동작합니다.
